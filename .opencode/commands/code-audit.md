Execute the 'code-audit' skill on the entire codebase.

1. Perform a thorough, senior-level analysis of the project structure, backend, and frontend code.
2. Specifically identify:
   - Security vulnerabilities.
   - Dead/unused code.
   - Poor implementation patterns (e.g., lack of error handling, hardcoding, tight coupling).
   - Incomplete functionality (TODOs or unimplemented logic).
3. For each finding, provide:
   - A clear description of the issue.
   - A severity score (1 = low, 5 = critical/urgent).
   - **A comprehensive, step-by-step mitigation and correction plan (including code snippets, configuration changes, or refactoring steps).**
4. Present the final output as a Senior Software Engineer, prioritizing critical (4-5) issues and categorizing them clearly for developer review, ensuring all mitigation plans adhere to project coding conventions.
