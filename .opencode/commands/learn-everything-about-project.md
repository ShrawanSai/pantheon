description: Analyzes project documentation and validates it against the actual codebase to produce a product-friendly status report highlighting requirements versus actual implementation.
---

Use the general agent to perform the following:

1. Scan all documentation (markdown files, PDFs, etc.) in the repository to extract business requirements and claimed features.
2. Analyze the codebase (apps/api/app, apps/web) to understand the actual implemented functionality.
3. Compare claimed features/requirements against the actual implementation.
4. Produce a status document suitable for a Product Manager:
   - Summarize business requirements.
   - Clearly outline implemented features.
   - Explicitly list missing features or discrepancies (claimed but not implemented).
   - Maintain a high-level business perspective, minimizing overly technical jargon.
